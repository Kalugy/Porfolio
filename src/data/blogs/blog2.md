This guide shows you how to create a mobile app using **Expo** and
upload it to the **Google Play Store** using the Expo build service.

------------------------------------------------------------------------

## 1. Install Expo

Install Expo CLI:

``` bash
npm install -g expo-cli
```

Create a new project:

``` bash
expo init my-app
```

Start the project:

``` bash
cd my-app
expo start
```

------------------------------------------------------------------------

## 2. Build Your App

Create basic screens using JavaScript or TypeScript inside the `App.js`
file:

``` jsx
import { Text, View } from "react-native";

export default function App() {
  return (
    <View style={{ flex: 1, justifyContent: "center", alignItems: "center" }}>
      <Text>Hello from Expo!</Text>
    </View>
  );
}
```

Test the app in Expo Go (Android/iOS).

------------------------------------------------------------------------

## 3. Prepare App for Production

Update `app.json` or `app.config.js`:

``` json
{
  "expo": {
    "name": "My App",
    "slug": "my-app",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "android": {
      "package": "com.myname.myapp"
    }
  }
}
```

------------------------------------------------------------------------

## 4. Build Release AAB With Expo

Expo handles everything:

``` bash
expo build:android
```

Or using EAS:

``` bash
eas build -p android --profile production
```

Expo generates an **AAB file** ready for Play Store upload.

------------------------------------------------------------------------

## 5. Upload to Play Store

1.  Go to [Google Play Console](https://play.google.com/console)
2.  Create a **new app**
3.  Fill:
    -   App name\
    -   Description\
    -   Screenshots\
    -   Icon\
    -   Privacy policy\
4.  Upload the AAB generated by Expo
5.  Complete:
    -   App content\
    -   Data safety\
    -   Target audience\
6.  Submit for review

------------------------------------------------------------------------
